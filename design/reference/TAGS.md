# Tag Reference

All tags used across the design documents. Use these in Obsidian's Graph View filters.

---

## Layer (Where it lives)

| Tag | Description |
|-----|-------------|
| `layer/ui` | User interface components |
| `layer/core` | Business logic, services |
| `layer/data` | Database, repositories |

---

## Domain (What it touches)

| Tag | Description |
|-----|-------------|
| `domain/audio` | Audio playback, files, metadata |
| `domain/database` | Schema, tables, migrations |
| `domain/playlist` | Playlist management, queue |
| `domain/table` | Library table view |
| `domain/tags` | Tag system, genres, categories |
| `domain/audit` | Logging, undo, history |

---

## Status (Where it's at)

| Tag | Description |
|-----|-------------|
| `status/done` | Completed |
| `status/active` | Currently in progress |
| `status/planned` | Designed, ready to implement |
| `status/future` | Far out, after other work |

---

## Type (What kind of doc)

| Tag | Description |
|-----|-------------|
| `type/feature` | End-user feature |
| `type/architecture` | System design |
| `type/widget` | Reusable UI component |
| `type/service` | Background service |
| `type/task` | Implementation checklist |
| `type/design` | Visual/UX design |

---

## Size (Effort estimate)

| Tag | Description |
|-----|-------------|
| `size/small` | 1-2 days |
| `size/medium` | 3-5 days |
| `size/large` | 1-2 weeks |
| `size/xlarge` | Multiple weeks |

---

## Special

| Tag | Description |
|-----|-------------|
| `priority/high` | Do this soon |
| `complexity/high` | Requires careful design |
| `safety/critical` | Data integrity concerns |
| `blocked/schema` | Waiting for DB changes |

---

## Value (Business/User Impact)

| Tag | Description |
|-----|-------------|
| `value/high` | Major user benefit |
| `value/medium` | Nice to have |
| `value/low` | Minor improvement |

---

## Risk (Bug/Stability Potential)

| Tag | Description |
|-----|-------------|
| `risk/high` | Could break things |
| `risk/medium` | Some caution needed |
| `risk/low` | Safe change |

---

## Scope (How Much It Touches)

| Tag | Description |
|-----|-------------|
| `scope/local` | Self-contained feature |
| `scope/system` | Touches many components |

---

## Needs (External Requirements)

| Tag | Description |
|-----|-------------|
| `needs/hardware` | Physical device integration |
| `needs/network` | Internet connectivity |
| `needs/api` | 3rd party API calls |
| `needs/shell` | System commands |

## Skill (Expertise Required)
